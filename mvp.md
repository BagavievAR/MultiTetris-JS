# Функциональные требования к MVP MultiTetris JS

## Минимально жизнеспособный продукт (MVP) должен реализовать следующий набор функций:

1. Игра (клиентская логика и визуализация)

- Работающий движок Тетриса (падение фигур, вращение, очистка строк)

- Игровой Canvas с отрисовкой текущего состояния

- Поддержка клавиатурного управления

- Подсчёт очков и отображение статистики (очки, уровень, время)

2. Соло-режим

- Доступ без авторизации

- Сессия заканчивается при проигрыше

- Очки отображаются по завершению игры

- При наличии аккаунта — сохраняются в историю и влияют на лидерборд

3. Мультиплеер (Score Battle)

- Выбор режима и создание комнаты

- Подключение по коду или в случайную открытую комнату

- Лобби ожидания (до начала игры)

- Одновременная игра с другими пользователями

- Обмен событиями по WebSocket (фигуры, очки, завершение игры)

- Выявление победителя по очкам

4. Аутентификация

- Регистрация с логином и паролем

- Вход с JWT-токеном и хранение сессии

- Гость также может играть (без сохранения статистики)

5. Профиль

- Отображение имени, аватара, ID

- История игр: дата, режим, результат, противники

- Список друзей (добавление/удаление)

- Настройки: звук, сетка, язык интерфейса

6. Лидерборд

- Топ игроков по очкам и победам

- Фильтры: по режиму (solo, score), по периоду (день/неделя/месяц)

- Глобальный и среди друзей

7. Локализация

- Переключение языка (RU/EN)

- Сохранение выбора (в localStorage или профиле)

- Интерфейс поддерживает оба языка во всех ключевых компонентах

8. Навигация и структура сайта

- Главная страница с выбором режима

- Страницы: Solo, Multiplayer, Room, Profile, Leaderboard, About, Help, Support

- 404-страница



# Приоритеты реализации для MVP

## Ядро (обязательно реализовать):

1. UC1: Соло-игра — позволяет пользователю начать игру сразу

2. UC2: Регистрация и вход — обеспечивает базовую авторизацию

3. UC3: Мультиплеер — основная фича проекта, реализует онлайн-игру

## Второстепенные (можно упростить или перенести на позже):

1. UC4: Приватные комнаты — реализация после общего мультиплеера

2. UC7: Лидерборд — для MVP может быть только глобальным, без фильтров

## Дополнительные:

1. UC1: После завершения игры в роли гостя можно предлагать регистрацию: “Хотите сохранить рекорд?”

2. UC3: Уточнить бизнес-правила:

- Минимум игроков для старта: 2

- Максимум: 4 (в рамках MVP)

3. UC3: Добавить обработку разрыва соединения:

- Если игрок отключается, его игра считается завершённой

- Он исключается из лидерборда текущей сессии



# Use Cases для MVP MultiTetris JS

## UC1. Начать соло-игру

Актор: Гость или авторизованный пользователь

Цель: Сыграть одиночную игру на очки

Сценарий:

- Пользователь заходит на главную

- Нажимает “Играть в одиночку”

- Загружается игровая сцена

- Пользователь играет до проигрыша

- Результат отображается, сохраняется (если авторизован)

## UC2. Зарегистрироваться и войти

Актор: Гость

Цель: Получить доступ к персональному профилю и статистике

Сценарий:

- Пользователь открывает /register

- Вводит логин, пароль, подтверждает

- Система создаёт аккаунт и авторизует

- Пользователь перенаправляется на главную или профиль

## UC3. Подключиться к мультиплееру (Score Battle)

Актор: Авторизованный пользователь

Цель: Принять участие в соревновательном режиме

Сценарий:

- Пользователь выбирает Score Battle на /multiplayer

- Нажимает "Найти матч" или присоединяется к комнате

- Ждёт игроков в лобби

- Игра запускается, очки обновляются в реальном времени

- Победитель определяется, результат сохраняется

## UC4. Создать приватную комнату

Актор: Авторизованный пользователь

Цель: Пригласить друга в закрытую игру

Сценарий:

- Игрок нажимает “Создать приватную комнату”

- Устанавливает пароль или выбирает "только для друзей"

- Получает код приглашения (/invite/:token)

- Отправляет его другу

- Второй игрок присоединяется, игра начинается

## UC5. Просмотреть профиль и историю

Актор: Авторизованный пользователь

Цель: Узнать свою статистику

Сценарий:

- Пользователь открывает /profile

- Видит имя, аватар, ID, список друзей

- В разделе "История игр" — список матчей с датами и результатами

- Может просмотреть детали игры (режим, очки, оппоненты)

## UC6. Переключить язык интерфейса

Актор: Любой пользователь

Цель: Использовать интерфейс на предпочитаемом языке

Сценарий:

- Пользователь кликает на иконку языка в Header

- Выбирает Русский или English

- Интерфейс мгновенно обновляется

- Выбор сохраняется в localStorage

## UC7. Просмотреть лидерборд

Актор: Любой пользователь

Цель: Ознакомиться с результатами других игроков

Сценарий:

- Открывает /leaderboard

- Выбирает режим и период (день/неделя/месяц)

- Смотрит глобальные результаты или только среди друзей

## UC8. Получить помощь

Актор: Любой пользователь

Цель: Понять правила и управление

Сценарий:

- Переходит на /help или нажимает ? в игре

- Видит подсказки: клавиши, цели, советы
