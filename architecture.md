## Компоненты по проектам

### Клиентское приложение (apps/client)

Layout.tsx - Обёртка для всех страниц, включает Header и Footer

Header / Footer - Навигация, язык, авторство

GameBoard - Основное игровое поле с фигурами (canvas)

ScorePanel - Очки, таймер, текущий уровень

PlayerPreview - Мини-просмотр полей других игроков (в мультиплеере)

ControlsOverlay - Подсказки по управлению (упавшая фигура, клавиши)

LoginForm / RegisterForm - Формы аутентификации пользователя

ProfileCard - Информация о текущем игроке (имя, аватар, ID)

HistoryList - История сыгранных игр и результатов

LeaderboardTable - Таблица лидеров (фильтрация по режимам/времени)

LanguageSwitcher - Компонент переключения языка интерфейса

Loader / Modal - Универсальные вспомогательные компоненты

audioManager.ts - Модуль централизованного воспроизведения звуков и музыки

useAudio.ts - React-хук для проигрывания звуков по событиям

### Серверное приложение (apps/server)

index.ts - Точка входа, запуск Socket.IO и Express сервера

sockets/ - Обработка соединений, событий и дисконнектов игроков

core/ - Бизнес-логика тетриса: падение фигур, проверка линий

rooms/ - Логика создания, хранения и синхронизации комнат

users/ - Аутентификация, друзья, история игр, бан-лист

leaderboard/ - Подсчёт очков и рейтингов, обработка результатов

utils/ - UUID генераторы, таймеры, логгеры и др. вспомогательные функции

### Общие типы и утилиты (packages/shared)

types/ - Общие интерфейсы: GameState, PlayerInfo, SocketEvents

constants/ - Константы для игры: размеры, уровни, фигуры, цвета

utils/ - Утилиты генерации фигур, проверки столкновений, ротации

## Используемые библиотеки

react (^18.2.0): UI-библиотека для компонентов. Используется для построения всех страниц и интерфейсов

react-dom (^18.2.0): Рендеринг React-компонентов в браузер

react-router-dom (^6.18.0): Навигация между страницами (роутинг)

typescript (^5.2.0): Статическая типизация. Используется во всём проекте

vite (^4.4.0): Сборка фронтенда и запуск dev-сервера

tailwindcss (^3.3.2): Быстрое оформление UI через utility-классы

i18next (^23.4.3): Переключение языков интерфейса и локализация

zustand (^4.4.1): Хранение глобального состояния. Используется в contexts/ — для хранения состояния авторизации, текущего игрока, выбранной комнаты и настроек игры.

axios (^1.4.0): HTTP-запросы с клиента на сервер (авторизация, лидерборд)

socket.io-client (^4.7.2): Подключение к WebSocket-серверу (игровые события в реальном времени)

express (^4.18.2): Серверная платформа на Node.js для REST и WebSocket

socket.io (^4.7.2): WebSocket-сервер для обмена событиями в мультиплеере

jsonwebtoken (^9.0.0): Подпись и проверка JWT токенов (аутентификация)

bcrypt (^5.1.0): Хэширование паролей при регистрации и входе

eslint (^8.56.0): Проверка кода на ошибки и соблюдение стиля

nodemon (^3.0.2): Автоперезапуск сервера при изменениях кода

dotenv (^16.3.1): Хранение конфигураций и секретов в .env

cors (^2.8.5): Разрешение CORS-запросов от клиента к серверу в dev-среде

jest (^29.7.0): Тестирование бизнес-логики (например, игровой движок, scoring)

@testing-library/react (^14.0.0): Тестирование UI-компонентов React

## Роутинг приложения

/ — Главная страница (выбор режима игры)

/solo — Соло-игра (одиночный тетрис на очки)

/multiplayer — Выбор режима мультиплеера

/multiplayer/score — Score Battle режим (несколько игроков, очки)

/multiplayer/royale — Battle Royale режим (PVP с отправкой линий)

/room/:mode/:id — Игровая комната (по ID и режиму)

Пример: /room/score/abc123

Включает /lobby как начальный экран, затем — саму игру

/invite/:token — Приватное приглашение в комнату (для друзей)

/login — Вход

/register — Регистрация

/profile — Профиль игрока (статистика, настройки, друзья)

/leaderboard — Таблица лидеров с фильтрами (по режиму, периоду)

/about — О проекте

/support — Поддержать проект

/help — Подсказки по управлению

(или /404) — Страница не найдена
